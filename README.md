# Cryptographic Algorithms Project

This repository contains implementations of several cryptographic mechanisms developed as part of a university cryptography project.

The project demonstrates how different cryptographic primitives can be combined to simulate a simplified blockchain-like structure.

## Implemented Components

### 1. GOST 34.11-2018 Hash Function

Implementation of the Russian cryptographic hash standard **GOST 34.11-2018** with a 256-bit hash output.

The hash function is used as the core primitive for other components of the project.

### 2. Pseudorandom Number Generator (PRNG)

A pseudorandom number generator based on iterative hashing using the implemented GOST hash function.

The generator works in cycles:

- Initial input: 512 bits containing the student's name
- Output: 256-bit hash value
- Subsequent values are generated using the previous hash and iteration counter

### 3. Schnorr Digital Signature

Implementation of the **Schnorr digital signature scheme** in a multiplicative group of a finite field.

Parameters are taken from **GOST R 34.10-94 test example**:

- prime modulus `p`
- subgroup order `q`
- generator `g`

The PRNG from the previous step is used to generate secret keys and nonces.

### 4. Transaction Signing

Five transactions (200 bytes each) are generated and signed using the Schnorr signature implementation.

Transactions are filled with pseudorandom data generated by the PRNG.

### 5. Merkle Tree

A Merkle tree is constructed using the hash function implementation.

The root hash represents the combined integrity of all signed transactions.

### 6. Block Header and Nonce Mining

A block header is constructed with the following fields:

- block size
- previous block hash
- Merkle root
- timestamp
- nonce

The program iteratively changes the nonce value to find a hash where the first **five bits are equal to zero**, simulating a simplified proof-of-work process.

## Technologies

The goal of the project is to gain practical experience with:

- hash functions
- pseudorandom number generation
- digital signatures
- Merkle trees
- simplified blockchain mechanics
